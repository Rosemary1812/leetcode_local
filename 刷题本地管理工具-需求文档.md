# 刷题本地管理工具 — 需求文档

> 本文档面向 AI 助手（Claude Code），用于在任意目录下构建一套完整的本地 LeetCode 刷题仓库。
> 无需 leetcode.com 账号，无需在线判题，完全离线运行。

***

## 项目概述

为技术面试刷题者提供一套本地题目管理工具，模拟真实面试环境，同时支持 Python 和 JavaScript 双语言。

### 目标

- 在本地管理 LeetCode 题目（编号、中文名、英文名、难度、描述）
- 支持 Python / JavaScript 双语言解答，放在同一文件夹下
- 通过脚本管理题目（批量创建、查看进度、快速打开）
- 通过 AI 辅助生成含测试集的刷题笔记

### 约束条件

- **不使用** leetcode.com / acm.com 等在线平台
- **不使用** 任何网站的辅助功能（自动补全、提交判题等）
- 题目描述嵌入 solution 文件头部注释
- solution 文件本身不引入任何外部依赖
- 支持完全离线使用

### 目标用户

- 需要准备技术面试
- 主要使用 Python 和 JavaScript 刷题
- 希望在本地模拟面试环境

***

## 题库范围

由用户提供题目列表文件（如 `高频.md`、`codetop.md` 等），解析其中的 LeetCode 题目编号，生成对应的文件夹。

默认推荐题库：**LeetCode Hot 100**（100 道）或**高频面试题**（40-50 道）。

***

## 目录结构

```
{项目根目录}/
├── high/                        # 题目存放目录（由题库文件名命名）
│   ├── {题号:03d}/              # 例：003、070、1143
│   │   ├── solution.py          # Python 题解
│   │   ├── solution.js          # JavaScript 题解
│   │   └── notes/               # AI 生成笔记（完成后）
│   │       └── {题号}-{英文标题}.md
│   └── ...
├── scripts/
│   ├── setup.py                # 批量创建所有题目文件夹（离线可用）
│   ├── fetch_descriptions.py   # 批量拉取题目描述（需联网一次）
│   ├── init_problem.py         # 初始化单个题目（需 leetcode-cli）
│   ├── list_problems.py        # 列出所有题目及完成状态
│   └── open_problem.py         # 快速打开题目文件夹
├── skill.md                    # AI 笔记生成器说明（AI 读取此文件）
├── requirements.txt             # Python 依赖
└── README.md                   # 使用说明
```

### 文件夹命名规范

- 格式：`{题号:03d}`，例如 `003`、`070`、`1143`
- 不包含题目名称，避免重命名导致的路径问题

***

## Solution 文件规范

### solution.py

```python
# ===== {题号}. {中文题目名} =====
# 难度: {难度等级}
# 英文名: {英文题目名}
# 来源: https://leetcode.cn/problems/{slug}/description/
#
# {题目描述正文，包含示例}
#
# ---------------------------------------------------------

class Solution:
    def method_name(self, ...):
        # TODO: implement
        pass
```

### solution.js

```javascript
// ===== {题号}. {中文题目名} =====
// 难度: {难度等级}
// 英文名: {英文题目名}
// 来源: https://leetcode.cn/problems/{slug}/description/
//
// {题目描述正文，包含示例}
//
// ---------------------------------------------------------

function methodName(...) {
    // TODO: implement
}
```

### solution 文件约定

- 题目描述位于文件头部注释，以 `#`（Python）或 `//`（JS）开头
- 代码框架位于分隔线 `# ---------------------------------------------------------` 之后
- 初始状态：`class Solution` / `function methodName` + `TODO: implement`
- 不使用任何外部库导入

## 工具脚本详细说明

### `setup.py`（离线可用）

批量创建所有题目文件夹，使用内置题库元数据（题号、中文名、英文名、难度）。不依赖 leetcode-cli，不联网。

**使用方法：**

```bash
python scripts/setup.py
```

**流程：**

1. 读取题库元数据（题号、中文名、英文名、难度）
2. 遍历每个题号，创建 `high/{题号:03d}/` 文件夹
3. 在文件夹内生成 `solution.py` 和 `solution.js`
4. 文件头部包含题号、中文名、难度、来源链接，题目描述为占位符

### `fetch_descriptions.py`（需联网一次）

从 GitHub 公开仓库（`azl397985856/leetcode`）批量抓取所有题目的中文题目描述，自动填充到 solution 文件头部注释中。**无需 leetcode-cli 登录**，仅需联网。

**使用方法：**

```bash
python scripts/fetch_descriptions.py
```

**流程：**

1. 读取 `setup.py` 中的题库元数据
2. 构造 GitHub Raw URL，从 azl397985856 仓库拉取 Markdown 格式题目描述
3. 解析 Markdown，提取 `## 题目描述` / `## Description` / `## Problem Description` 段落
4. 对于无法从仓库获取的题目，使用内置硬编码描述作为备选
5. 将描述填充到已有的 `solution.py` 和 `solution.js` 文件头部

**数据源：**

- 主数据源：`https://raw.githubusercontent.com/azl397985856/leetcode/master/problems/{文件名}.md`
- 内置备选：当 GitHub 仓库找不到时，使用脚本中硬编码的题目描述

### `init_problem.py`（需 leetcode-cli）

根据题目编号初始化单个题目。需要已安装并登录 leetcode-cli。

**使用方法：**

```bash
python scripts/init_problem.py 1
python scripts/init_problem.py 1 2 15 20
python scripts/init_problem.py 1 --force   # 强制覆盖
```

**流程：**

1. 查找 leetcode-cli 缓存目录（Windows: `%APPDATA%\leetcode-cli\data`）
2. 缓存命中：直接读取 JSON 文件
3. 缓存未命中：调用 `leetcode download {id}` 联网下载
4. 解析 JSON，提取题目描述（HTML 转纯文本）
5. 生成 `solution.py` 和 `solution.js`
6. 若文件夹已存在，提示用户确认是否覆盖

### `list_problems.py`（离线可用）

列出所有已创建的题目，显示编号、名称、难度、完成状态。

**使用方法：**

```bash
python scripts/list_problems.py
```

**输出示例：**

```
  编号  题目名称                       难度   状态
-------------------------------------------------------
  003   无重复字符的最长子串          中等   [OK]
  005   最长回文子串                中等   [--]
  070   爬楼梯                      简单   [OK]
  072   编辑距离                    困难   [--]
-------------------------------------------------------
已完成: 2 / 4
```

**"已完成"判断逻辑：**

- `solution.py` 或 `solution.js` 中存在非 TODO、非 pass 的实际代码行

### `open_problem.py`（离线可用）

快速打开指定题目的文件夹（Windows 资源管理器）。

**使用方法：**

```bash
python scripts/open_problem.py 70
```

### `requirements.txt`

```
leetcode-cli
```

（leetcode-cli 作为 npm 包安装：`npm install -g leetcode-cli`，不在 pip 依赖中）

## AI 笔记生成器

详见同目录下的 `skill.md`。

触发方式：用户使用 `/skill {题号}` 或直接要求 AI 生成某道题的笔记。

### 执行流程

1. AI 读取 `skill.md`，理解笔记格式要求
2. 读取 `high/{题号:03d}/solution.py` 获取用户代码
3. 从 solution.py 头部注释读取题目描述
4. 判断用户是否提供了题解：
   - **有题解**：保留原有逻辑，整理格式、补充注释，不替换写法
   - **无题解**：生成最简洁的参考解法
5. 根据题目描述构造测试集（基础用例 + 边界用例，2-5 条）
6. 生成完整 Markdown 笔记，包含：
   - 题目描述
   - 思路分析（核心方法 + 2-4 句说明）
   - 复杂度分析（时间 + 空间，含推导过程）
   - 测试集（基础 / 边界 / 扩展）
   - JavaScript 题解
   - Python 题解
   - 本地测试脚本 `tests/test_{题号}.py`
7. 保存至 `high/{题号:03d}/notes/{题号}-{英文标题}.md`

## 使用流程（用户视角）

### 首次设置

```bash
# 1. 安装 leetcode-cli（用于单个题目初始化，可选）
npm install -g leetcode-cli

# 2. 批量创建所有题目文件夹
python scripts/setup.py

# 3. 联网拉取所有题目描述（仅需一次）
python scripts/fetch_descriptions.py
```

### 日常刷题

```bash
# 打开某道题
python scripts/open_problem.py 70

# 查看进度
python scripts/list_problems.py

# 刷完一道题后，让 AI 生成笔记
# 对 AI 说：/skill 70
# 或："生成第 70 题的笔记"
```

## 边界情况处理

| 场景               | 处理方式                                                 |
| ---------------- | ---------------------------------------------------- |
| 题目编号不存在          | 提示"题目不存在"                                            |
| 文件夹已存在           | `setup.py` 跳过；`init_problem.py` 提示"是否覆盖？\[y/N]"      |
| leetcode-cli 未安装 | `init_problem.py` 提示安装命令                             |
| GitHub 拉取失败      | 使用内置硬编码描述兜底                                          |
| 网络断开             | `fetch_descriptions.py` 失败，使用已有的 `setup.py` 骨架继续离线使用 |

## 扩展说明

### 更换题库

修改 `setup.py` 中的 `BUILTIN_METADATA` 字典即可切换题库。字典格式：

```python
{
    题号: {
        "title_cn": "中文名",
        "title_en": "English Name",
        "difficulty": "简单/中等/困难",
    }
}
```

### 题目来源

推荐使用 `azl397985856/leetcode` 仓库的 `problems/` 目录下的 Markdown 文件作为描述来源。该仓库题目覆盖全面，中文描述质量高，且无需登录。

### 笔记目录

`notes/` 子目录在 `setup.py` 创建文件夹时**不预建**，由 AI 在生成笔记时按需创建。
